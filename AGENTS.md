# AGENTS.md

This file provides guidance to AI coding agents when working with code in this repository.

## Build & Run

```bash
flutter pub get                        # Install dependencies
flutter pub run build_runner build --delete-conflicting-outputs  # REQUIRED before compile/test
flutter run                            # Run on default device
flutter analyze                        # Lint
flutter test                           # Run all tests
dart format .                          # Format code
```

**Code generation is mandatory.** The `build_runner build` step generates Hive type adapters (`*.g.dart`), JSON serialization, and the `env.g.dart` secrets file. The app will not compile without it.

## Required Environment

A `.env` file must exist in the project root with:

```
FDC_API_KEY=...
SENTRY_DNS=...
SUPABASE_PROJECT_URL=...
SUPABASE_PROJECT_ANON_KEY=...
```

These are consumed by `envied` during code generation. Without them, `build_runner` fails. In CI, they are injected as GitHub Actions secrets.

Flutter version is pinned to **3.27.1**.

## Architecture

**Clean Architecture + BLoC**, organized by feature under `lib/features/`:

- **Presentation:** `flutter_bloc` — each feature has `presentation/bloc/` with `*_bloc.dart`, `*_event.dart`, `*_state.dart` (using `part` files)
- **Domain:** Use cases and entity classes in `domain/usecase/` and `domain/entity/`
- **Data:** Repositories wrapping data sources. Local storage uses **Hive** (encrypted via `flutter_secure_storage`). Remote: Open Food Facts API, USDA FDC API, Supabase
- **DI:** `get_it` service locator in `lib/core/utils/locator.dart` — BLoCs as `registerFactory`, everything else as `registerLazySingleton`
- **Theme state:** Uses `provider` (`ThemeModeProvider`) — this is the only use of Provider; all other state uses BLoC

Shared code lives in `lib/core/` (data sources, entities, use cases, widgets, styles, utils).

## Localization

Uses `flutter_intl` plugin. ARB files in `lib/l10n/` (en, de, tr). Generated code in `lib/generated/` — do not edit generated files. Access strings via `S.of(context).someKey`.

## Code Style

- Standard `dart format` (2-space indent)
- `flutter_lints` with no custom overrides
- Codebase uses double quotes
- BLoC states/events extend `Equatable`

## Commits

Use conventional commits: `feat:`, `fix:`, `chore:`, `refactor:`, `test:`, `docs:`, etc.

## Don't Edit

- `lib/generated/` — auto-generated localization
- `*.g.dart` files — auto-generated by `build_runner`
- `.env` — contains secrets
